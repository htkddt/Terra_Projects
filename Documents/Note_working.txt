		+ The root cause of this ticket is due to missing whitespace between elements when running list_node_regbus_layers -tcl.
		+ Before GUI prints output in the console, data will be processed in the to_formatted_stream() function at /noc_dev/Infra/src/G.cpp:1488
		+ In the to_formatted_stream() function, data will be cut from a long line into many short lines by the transpose_move() function.
		+ The transpose_move() function will return a vector which contains the short lines. If data is missed whitespace, that function will return a vector with the size of vector is 1. That means GUI will print a very long 
		line in the console which will get stuck.
		+ If add the whitespace between elements when running list_node_regbus_layers -tcl, the output in the console will be similar when running that command without the -tcl option. That is also the problem of Yashwanth C S.
		
		
		From this point, there will two options according to @Yashwanth C S requirement
			1# The console will display similar the new format. (Recommend)
			2# If you want the console display similar the old format, GUI will be stuck with a large number of elements.
